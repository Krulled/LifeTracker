How should nutrition data be extracted from the AI reply?

The chat endpoint currently returns plain text. We need structured data (food_name, calories, protein_g, carbs_g, fat_g) for the draft buttons.

A

Single LLM call — embedded JSON block (Recommended)

Instruct the AI to append a machine-readable block at the end of its reply. Backend strips it, returns clean reply text + drafts array. One round-trip, no extra latency.

Pros

  • Single LLM call — same speed as today
  • No backend refactor needed
  • llama-3.1-8b follows structured output well when prompted

Cons

  • Need robust parsing (regex) to strip JSON from reply
  • Occasional LLM formatting slips
B

Two-phase — reply then extract

First call generates the natural language reply as today. A second, tiny LLM call then extracts structured items from that text. Clean separation of concerns.

Pros

  • Reply is guaranteed clean text
  • Extraction prompt is small and focused

Cons

  • 2× LLM calls → more latency per message
  • Added complexity in backend
C

Frontend regex parsing — no backend changes

Parse the plain text reply in the browser using regex patterns to find calorie/macro numbers. No backend changes at all.

Pros

  • Zero backend changes
  • Instant to ship

Cons

  • Fragile — breaks on varied phrasing
  • Hard to reliably identify food name vs calorie numbers
  • Poor accuracy for multi-food replies

Approach A is recommended — single call, minimal latency, reliable enough with a clear prompt contract. Click to select or reply in terminal.
