How should the scan feature branch between a food photo and a nutrition label?

Currently there's one "📷 Scan" button. We need to handle two very different inputs.

📷 Scan
📋 Templates
📷 Scan Food
🍽️ / 📋
Take any photo — AI detects type
📁 Gallery
📷 Camera
🤖 AI auto-detects: meal or nutrition label

A — Auto-detect

One "Scan" button. AI figures out if the photo is a plated meal or a nutrition label and branches the flow automatically.

Pros

  • Simplest UX — one button
  • Works even if user is unsure

Cons

  • AI might misclassify
  • Extra round-trip before branching

B — Two buttons

Add a separate "🏷️ Scan Label" button alongside the existing "📷 Scan Food". Each opens a dedicated modal with purpose-built UI.

Pros

  • Zero ambiguity — user declares intent
  • Cleaner per-mode prompts

Cons

  • Two buttons to explain
  • Slightly more crowded toolbar

C — Tab toggle inside modal

One "Scan" button opens a modal with a "Meal Photo / Nutrition Label" tab toggle at the top. User switches mode before uploading.

Pros

  • One entry point, clear mode switch
  • UI stays compact on main screen

Cons

  • One extra tap vs. two dedicated buttons
